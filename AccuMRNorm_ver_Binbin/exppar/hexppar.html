What is "event" and "parameters" in data ?


    We use task control program (essXXXX) running on a QNX machine to record events of stimlus timing, subjects responses etc.  A way of recording is dependent on implementaton in the conrol program.  Our standard ess system is "MriGeneric".  Once data files are opened from ESSGUI, .evt (raw event file before data compilatoin/conversion) will be created.  In the case of our standard ess system, "MriGeneric", it normally communicates with the stimulus computer and stimulus control program (MriStim) and stimulus parameter files (.stm, .pdm and .hst) will be dumped.  If the flag in ESSGUI is set corectly, analog data files (.adf or .adfw) will also be opened.  After the end of experiments, those .evt should be compiled with eye movement and/or vital signals into .dgz.  For mri parameters see hmri for detail.
    Most of experiment parametes will be save as "SesPar.mat" by
>> sesdumppar(SESSION)
Parameters can be accessed like
>> par = expgetpar(SESSION, ExpNo)
par =
      evt: [1x1 struct]    <--- event 
    pvpar: [1x1 struct]  <--- MRI
      adf: [1x1 struct]    <--- adf/adfw
      stm: [1x1 struct]    <--- stimulus parameters, only if "SESSION/stmfiles/xxxx.stm" exists.
      rfp: [1x1 struct]     <--- receptive field parameters, only if "SESSION/stmfiles/SESSION.rfp" exits.

Usually .evt and .pvpar and .stm are usuful for data analysis.
For an example, (m02lx1, Exp=1),  .evt.obs has event timings and parameters for all observation periods as a cell array.
par.evt =

                prmnames: {}

               validobsp: 1

Stimulus information is like
par.stm =
      labels: {'obsp1'}
     ntrials: 1
    stmtypes: {'blank'  'movie'  'blank'}
       voldt: 0.2500                       <---  volume TR in seconds
           ========= FOLLOWINGS ARE CELL ARRAY OF EACH OBSERVATIO PERIODS ======
           v: {[0 1 2]}                    < ---  stimulus IDs for .stmtypes, Note to add +1 for matlab indexing.
          dt: {[60 300 30]}                <--- expected stimulus durations in seonds
           t: {[0 60 360 390]}             <--- expected) timing in seconds
        tvol: {[0 240 1440 1560]}          <---- expected iming in volumes
        time: {[0.0010 60.0052 360.0193]}  <--- actual timing in seconds
          ======================================================================
        date: 'Tue Aug 26 14:57:46 2003'
     stmpars: [1x1 struct]
     pdmpars: [1x1 struct]
     hstpars: [1x1 struct]

As you see above, "MriGeneric" is a single observation period system not to miss any MRI trigger events (par.evt.obs , par.stm.v/dt/t/tvol/time is a single cell array).  For convinience, recorded (not expected) stimlus timing is accessible as par.stm.time while it originally comes from event data (par.evt.obs).  To see indivisual stimulus parameters, look at par.stm.stmpars.stmobj.  For an example, parameters for stimulus ID of "1" (par.stm.stmypes{1+1} = 'movie'), see par.stm.stmpars.stmobj{1+1}.

Experiment Parameters

expgetpar (Ses, ExpNo) Returns experiment parameters, evt, pvpar and stm (See sesdumppar)
expgetstm (Ses, ExpNo) Provide stimulus-related information (stm, epoch, boxcar, hemo)
sesdumppar (Ses, Exps/GrpName) Get/Load all experimental parameteters of the session.
expgetevt (Ses, ExpNo) Uses adf_info/dg_read to get all events of experiment ExpNo
dumpdgz (dgzfile) print event information in given trials of DGZ.
gui/ dgzviewer (Ses, ExpNo)
gui/ dgzviewer (dgzfile)		 GUI utility to view the DGZ file
jpcode/ getpvpars (Ses, ExpNo) Get Reco/Acqp Paravision-Parameters for defined scan
utils/mex_dg/ dg_read (dgzfile) MEX utility to read the event (.dgz) file.
utils/mex_adf/ adf_info (adffile) MEX utility to get info of ADF/ADFW file.
