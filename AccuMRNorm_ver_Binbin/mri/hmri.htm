MRI Data: Initial Paravision Format and Directory Structures

Data are from anesthetized or alert monkey experiments. They are collected at the 4.7T or 7T scanners using the Paravision software. Paravision saves the data in their so-called FID (free induction decay) files as sequential echoes. The order of the Echo trains depends on image segmentation and the used protocol. We have a number of dynamic off-resonance correction algorithms that were developed by Josef Pfeuffer, which I leave (at this stage) out of description. I supposed (this will change in the near future) that all FID-file corrections were done by those understanding the details of the procedure and the typical user will be working with the so-called 2dseq files, which are reconstructed data generated after resorting and assembling different segments.

Most data are saved on the Wks8 data-server, in the directory (\\Wks8\Guest\nmr). The structure of the Paravision directory is

Z:\A00.3c1\1\pdata\1 (where Z is the mapped drive for \\Wks8\Guest\nmr)

Relevant to the analysis functions are the files 2dseq and Reco, whereby the latter contains information regarding the acquired images. In certain cases, also the file Acqp in the parent directory (see below) is used to obtain information on additional parameters.

Z:\A00.3c1\1

All the imaging parameters are accessed by invoking the getpvpars function which reads Acqp and Reco and picks the relevant variables. To ensure portability, i.e. to enable analysis of data on a laptop not connected to the network, a utility was developed that extracts all relevant physiology and MRI parameters/variables and subsequently stores them in a Mat file (SesPar.mat) in the session’s home directory (see below). Once this file is created analysis can be performed with no direct access to Wks8 or other servers with physiology data.

First Preparatory Steps for Data Analysis

Handling the raw data

All people interested in using efficiently this software should collect event files (DGZ files) during the data acquisition. Consult Mark Augath, Yusuke Murayama or myself to ensure you are using the right software and you have collected all event information that facilitates your data analysis. If no event files exist you must enter manually information about the main events (e.g. stimulus onset/offset) into the description file (see below). If physiology data are collected you must ensure that your ADF or ADFW files are saved on the appropriate server. At the end of the session the DGZ/ADF(W) files must will be converted with the convasdfw.ksh shell-script.

#!/usr/bin/sh

#

# FILE - convadfw.ksh

#

# PURPOSE - Script to allow multiple data files to be converted using cnvadfw

#

adfdir="//MyComputer/DataNeuro/MyDirectly"

cvtdir="//MyDisk/DataNeuro/MyDirectory/cvt"

Checking the Cardiac and Respiratory Signals

If you remember (you should!!) to collect the event files, you can use the sesvital function to create the Vital.mat file, which includes two signals: (a) a fast changing that is the plethysmogramm of the animal, that is the change of oxygen saturation over time. You expect this to be a low-pass filtered version of the EKG signal, and it should have an average frequency between 2 and 2.7Hz. Below 2 you have bradycardia, above 2.7 the techs are screwing badly. The lower-frequency signal is the output of the respirator. Please note that the pleth signal is composed by both respiration (modulatory low-frequency signal) and cardiac signal. Once the Vital.mat is created you can see the signals of an experiment or the entire session by typing showvital with one or two arguments. showvital (Session Name) will generate a 3D plot with all the session’s data (see below).

Showvital (SesName, ExpNo) will plot the signals of experiment ExpNo

Loading MRI Data

Loading Files: Setting Default Parameters

The loading of the images (2dseq files) is done by invoking the sesimgload function. This in turn calls imgload which does the actual reading by using the read2dseq function. The data read by these functions are saved under the session-directory (e.g. J02.x31), in the SIGS subdirectory. Similarly the anatomical and functional control scans can be read by calling the sesascan and sescscan functions.

The function sesimgload uses a number of parameters that determine possible image processing steps to carry out after loading the images. They include normalization, filtering, artifact removal, etc. The selection of options is entirely project-dependent. Eventually we must generate project-specific parameter files to avoid having several options of the same function. Currently, however, we go around the problem of diversity by generating project specific sesimgload functions, e.g. gpimgload for the Glass Pattern project. The parameters to be defined are:

DEFARGS.ISCANTYPE             = 'EPI';          % All MRI+Phis is EPI (one/multi shot)

DEFARGS.IDATATYPE             = 'tcImg';        % Usually the name of the structure

DEFARGS.ICLIP                 = [0.1 0.9];      % Clipping

DEFARGS.IGAMMA                = 0.8;            % Gamma value

DEFARGS.IRESP_FREQ            = 0.4;            % (Hz) 25 strokes / min

DEFARGS.ISUBSTITUDE           = 0;              % Substitude initial images to avoid trans

DEFARGS.ICROP                 = 1;              % Crop images

DEFARGS.IADJUST               = 0;              % Permit gamma/clip

DEFARGS.INORMALIZE            = 1;              % Ratio normalization

DEFARGS.INORMALIZE_THR        = 10;             % Percent of max to include in normaliz.

DEFARGS.IDETREND              = 0;              % Linear detrending

DEFARGS.IDETREND_AND_DENOISE = 0;               % Detrend and remove resp artifacts

DEFARGS.ITMPFLT_LOW           = 0;              % Reduce samp. rate by this factor

DEFARGS.ITMPFLT_HIGH          = 0;              % Remove slow oscillations

DEFARGS.IDENOISE              = 0;              % Remove respiratory art. (not used)

DEFARGS.IFILTER               = 0;              % Filter w/ a small kernel

DEFARGS.IFILTER_KSIZE         = 3;              % Kernel size

DEFARGS.IFILTER_SD            = 1;              % SD (if half about 90% of flt in kernel)

DEFARGS.ISAVE                 = 1;

DEFARGS.SAVEAS_IMG            = 0;              % tcImg.dat will be saved separately.

 

The fields shown in “bold” are usually selected to ensure data reduction, common grand-mean, linear trend removal, and some filtering for improving SNR, respectively.

Data Directories and Structures

Following the loading of the images into the corresponding experiment-based Mat file, the data can be examined by directly loading the Mat file using the appropriate pathname,

Load //wks20/DataMatlab/J02.x31/j02x31_001_TCIMG.mat;

Or by calling the sigload function

Sigload (‘j02x31’, 1,’tcImg’);

The tcImg structure contains the following fields:

Older version example:

   tcImg =

    session: 'j02x31'

    grpname: 'gpatcc1'

      ExpNo: 1

        dir: [1x1 struct]

        dsp: [1x1 struct]

        usr: [1x1 struct]

        dat: [4-D double]

         ds: [0.7500 0.7500 2]

         dx: 1

        grp: [1x1 struct]

        evt: [1x1 struct]

        stm: [1x1 struct]

The stm field contains the stimulus information and is essential for running group statistics, correlation analysis or multiple regression analysis. If no event files were collected, then the description file must be edited and the following fields must be defined in the group-structures:

      GRPP.v           = {[0 1 2 0]};

      GRPP.t            = {[20 44 44 20]};

GRPP.stmtypes     = {'blank','Conc-w0','Conc-w0','blank'};

The v-field shows the stimulus state, whereby 0 is off and the non-zero entries show the stimulus ID. The numbers 20, 44, 44, and 20 show the number of images per epoch. They are internally converted into time (seconds). The field stmtypes shows the stimulus-type, here blank, concentric random dots with no “wondering”, etc.

Creating of Regions of Interest (ROI)

Before continuing with the actual data analysis (e.g. correlation analysis, GLM etc.) the regions of interest ROIs must be defined by the user. The functions mroi or sesroi can be used to define ROIs.  The definable ROIs given in the description file:

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

% ROI for MRI experiments, if exist or needed, then put here

%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%%

ROI.groups              = {'all'};

ROI.names               = {'Brain','V1','V2','XS'}; XS means extrastriate…

 

To obtain help for the mroi program type hroi in the Matlab command window.

Once the ROIs are defined one can start the actual analysis of the data. If you have no reasons to define individual ROIs, you should at least define a brain ROI, to save computational time and enable the usage of the utilities extracting the time series of voxels (e.g. mareats).

Synopsis of Preparatory Steps

1.      Create the session's description file

2.      Generate the SesPar.mat file with all physiology/MRI parameters

3.      Check the preprocessing parameters in the sesimgload function. If necessary create a project-specific loading function (e.g. gpimgload instead of sesimgload).

4.      Load all images by using sesimgload (or the project specific loader)

5.      Load all anatomical and control (EPI13) scans

6.      Select ROIs

Extracting Time Series for Correlation/Multiple Regression Analysis

Extracting time series requires the definition of parameters in the description file that determine the signal-conditioning desired for a specific project.

      ANAP.mareats.IEXCLUDE       = {'V1';'V2';'XS'};     Exclude in MAREATS

      ANAP.mareats.ICONCAT        = 1;             1= concatanate ROIs before creating roiTs

      ANAP.mareats.IFFTFLT        = 0;

      ANAP.mareats.IARTHURFLT     = 0;

      ANAP.mareats.IMIMGPRO       = 0;

      ANAP.mareats.ICUTOFF        = 0;

      ANAP.mareats.ICUTOFFHIGH    = 0;

      ANAP.mareats.ICORANA        = 1;

      ANAP.mareats.ITOSDU         = 2;

      ANAP.mareats.IPLOT          = 0;

 

The removal of respiratory artifacts is done by the functions matsart and matsfft. The field ITOSDU shows the scale and baseline of the signal and it is used by the function xform (see input arguments). ITOSDU = 1 means, subtract the mean value of the baseline and divide by the baseline’s SD. ITOSDU means to the first thing only.

ANAP.Quality                = 80;    Percent (all exps good activation)

      ANAP.ImgDistort             = 1;     EPI-Ana can't be regist. due2distortions

      ANAP.PreTime                = 16;    Allow pre time when selecting stim for sigsort

      ANAP.PostTime               = 36;    Allow post time when selecting stim for sigsort

      ANAP.aval                   = 0.01;  Type II error for the statistical tests

      ANAP.bonferroni             = 0;     Bonferroni Correction

 

Please note, that the PreTime and PostTime are fields that are only used in experiments, in which the observation period has multiple trials. When you convert the long observation period into trials, then you can select some time before and after the trigger event (e.g. stimulus onset) to see what’s going on. The fields aval and Bonferroni are self-explanatory.

To extract time series we must call sesareats(Session-Name). This will subsequently call the function mareats that does the following things:

%MAREATS - Select and process the time series of selected ROIs or Areas

% MAREATS (SESSION, ExpNo, ARGS) extracts the time series for each of the permissible ROIs

% defined with the sesroi(SesName) utility.  The areas noted as "excluded"

% (e.g. IEXCLUDE={'brain';'ele'}) are not taken into account.

% 

% The selection of time series depends on individual projects. There are two general

% categories: (a) Projects in which the voxel identity is important; representative example is

% the "dynamic connectivity" project, in which dependence measures are determining the strength

% of connectivity between brain sites, and (b) Projects in which the exact voxel identity is

% irrelevant; an example, here, is the study of BOLD nonlinearities during brief-pulse

% stimulation. In this project the response of a region or area (e.g. V1, ele) is averaged

% across all voxels in the area, and this average response is compared with neural activity and

% so on and so forth.

%

% Both the IEXCLUDE and the ICONCAT fields will be different in the two types of project. To

% avoid editing this function or passing long lists of arguments, the IEXCLUDE, ICONCAT,

% etc. may be defined in the description file, as follows:

%

% DYNAMIC CONNECTIVITY

%       ANAP.mareats.IEXCLUDE = {'brain';'ele';};

%       ANAP.mareats.ICONCAT  = 0;

% BRIEF PULSES

%       ANAP.mareats.IEXCLUDE = {'brain';'ele';'test'};

%       ANAP.mareats.ICONCAT  = 1;

%

% The 'test' ROI is not needed for the second type of projects. It is used in the dependence

% studies to denote an area that is certainly not modulated by the stimulus.

%

% The ROIs are assumed to be already determined in the Ses.roi structure, including the

% following fields:

%       ROI.groups = {'all'};

%       ROI.names = {'brain';'v1';'v2';'test'};

%       ROI.model = 'v1';

% For details regarding these definitions type "hroi" To define the actual ROIs type

% "sesroi(SesName)".

%

% In the first category of projects, MAREATS returns a cell array which has the time series of

% each permissible (not excluded) ROI as defined by sesroi. This means, that multiple ROIs in a

% single cortical area will appear as different members of the cell array. This arrangments

% ensure the preservation of the convexity-constraint for ROI definition, which in turn

% warrants a Euclidean distance definition that corresponds as closely as possible to axonal

% distance (geodesic).

%

% In the second category MAREATS returns a cell array, each member of which is a cortical area

% as defined in the Ses.roi.names, barring the exclude areas.

%

% Note that the output of the MAREATS in the first project-category can be still concatanated

% by using the MROITSCAT (see below);

% 

% See also SESROI SESTCIMG MROI MROIGET MTCFROMCOORDS MROICAT MROITSCAT

%

% MTIMESERIES - Function to obtain the time series of voxels of a ROI Usage: MTIMESERIES

% (tcImg, Roi, RoiName) uses the ROI information in the structure Roi and select time series

% for each defined area or subregion in the structure;

%

% MROICAT - Concatanate all rois of the same area in a slice Usage: MyRoi = MROICAT (MyRoi);

% If, for example, we have right/left "V1", this operation will make the separate "V1" (left)

% and "V1" (right) one single area by or-ing the masks and concatanating the coordinates Called

% by: MTIMESERIES

%

% MROIGET - Select one area with one or multiple concatanated rois Usage: oRoi =

% mroiget(oRoi,[],RoiName); Called by: MTIMESERIES

%

% MTCFROMCOORDS - Get the time course of the signals for each voxel Usage:

% tc=mtcfromcoords(tcImg,coords); The function returns the time series of the voxels having

% coordinates "coords"; The coordinates are defined in the "save" case of the Main_Callback of

% the MROIGUI script; The tc (time courses) are processed according to the switches defined in

% "ARGS" Called by: MTIMESERIES

%

% MROITSCAT - Concatanates the ROI-based roiTs cell array members Usage: roiTs = MROITSCAT

% (roiTs);

%

% MATSCOR - Compute correlation coefficients for model mdlsct roiTs = matscor(roiTs,mdlsct) can
